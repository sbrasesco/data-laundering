Aignition Data Laundering — Reporte de Arquitectura

Descripción completa de la base de datos, el frontend React y el flujo de automatización n8n para el sistema de digitalización de comprobantes PDF → Excel.

Frontend: React 18 + Vite + TypeScript
DB: Supabase (PostgreSQL)
Automatización: n8n (40 nodos)
IA: Mistral OCR + OpenAI GPT
Generado: Mayo 2026
🏗️

1. Visión General

Arquitectura end-to-end del sistema

Flujo de datos principal: El usuario sube un ZIP desde el frontend → se crea el registro en pdf_jobs → se llama al webhook de n8n de forma asíncrona → n8n procesa los PDFs/imágenes con IA y escribe resultados en Supabase → el frontend se actualiza en tiempo real vía Realtime WebSockets.

🪝 Custom Hooks de datos
  • usePdfJobs
    Realtime Polling 5s
    Carga todos los jobs de la organización con conteo de filas. Suscripción Realtime a pdf_jobs y pdf_job_rows. Polling automático cada 5 segundos cuando hay jobs activos (últimos 15 min).
  • usePdfJob
    Realtime
    Detalle de un job específico. Suscripción Realtime filtrada por id=eq.{jobId} para actualizaciones en tiempo real del estado.
  • usePdfJobRows
    Filas extraídas de un job específico, ordenadas por created_at. Carga estática (sin Realtime en la versión actual).
  • useClientJobs
    Realtime RLS
    Jobs filtrados por cliente, rango de fechas. Calcula métricas agregadas. Suscripción Realtime con lógica de INSERT/UPDATE/DELETE diferenciada.
  • useClients
    RLS
    CRUD completo de clientes. Expone createClient, updateClient, toggleClientActive. Aplica organization_id desde el contexto de auth.
  • useActiveClients
    Clientes activos simplificados (id, name, tax_id). Usados en el selector del formulario de carga.
  • useAllDocuments
    Todos los documentos (pdf_job_rows) con join a pdf_jobs y clients. Paginación server-side, filtros por fecha y cliente. Búsqueda de texto client-side.
🔐 Flujo de autenticación
AuthContext gestiona la sesión globalmente con supabase.auth.getSession() y onAuthStateChange().
Al autenticarse, se consulta profiles para obtener el organization_id del usuario.
ProtectedRoute envuelve todas las rutas privadas y redirige a /login si no hay sesión.
El organization_id se propaga via contexto y se usa en todos los hooks para el aislamiento RLS.
📤 Flujo de carga de archivo (SubirZipPage)
1️⃣ Usuario selecciona ZIP + cliente + período
→
2️⃣ INSERT pdf_jobs (status: processing)
→
3️⃣ POST webhook n8n (background, no await)
→
4️⃣ Navigate → /dashboard
La llamada al webhook de n8n se hace con uploadFileToN8n(...).catch() — el frontend no espera la respuesta. El dashboard se actualiza en tiempo real vía Realtime cuando n8n va escribiendo en la DB.
📊 Exportación de datos
La página Documents permite exportar los documentos filtrados en dos formatos:
exportDocumentsToXlsx() — genera un archivo .xlsx usando la librería xlsx con todos los campos del comprobante.
exportToCSV() — genera un .csv plano para compatibilidad general.
Ambas funciones se ejecutan completamente en el cliente (browser).

🤖 IA: Prompt del AI Agent (OpenAI)
El sistema prompt instruye al modelo a actuar como extractor de comprobantes argentinos.

Reglas clave del prompt:
• Devolver solo JSON válido, sin texto extra
• Fechas en formato DD-MM-YYYY, importes como número
• Distingue EMISOR (encabezado del comprobante) vs RECEPTOR (identificado por etiquetas como "Señor/es:", "Cliente:", "A:")
• Ancla de receptor: si el frontend pasa un CUIT confirmado, ese CUIT ES el receptor con certeza absoluta
• Infiere tipo de letra (A/B/C) del título o número de comprobante
• Moneda: USD si aparece USD/U$S/Dólares, sino ARS
• Retorna confidence_score (0–1) por documento
🗺️ Nodo especial: Leer OC Map
El nodo Leer OC Map es un Code Node que recorre el directorio de trabajo y parsea los PDFs adjuntos del ZIP para extraer el mapeo entre archivos PDF y Órdenes de Compra (OC).

Técnica: extrae nombres de archivo desde el stream binario del PDF buscando las claves /F, /UF en formato raw. Soporta encoding UTF-16BE.

Resultado: un objeto oc_map { "archivo.pdf": ["OC-001", "OC-002"] } que luego se inyecta en el contexto de cada documento para anclar las OC en el AI Agent.
Tipos de documento soportados: FACTURA_A/B/C, NOTA_DEBITO_A/B/C, NOTA_CREDITO_A/B/C, ORDEN_COMPRA, SOLICITUD_COTIZACION. El flujo procesa PDFs con texto nativo (Extract from File) y PDFs escaneados / imágenes JPG/PNG (Mistral AI Vision). Ambas rutas convergen antes del AI Agent.
